# ⏱️ TimeTrackerApp

**TimeTrackerApp** — це сучасний, високопродуктивний та ергономічний десктопний застосунок для управління часом та завданнями, побудований на базі **WPF (C#)** та **SQLite**. Застосунок розроблено з використанням архітектурного патерну **MVVM**, що гарантує високу підтримуваність, чистоту коду та надійність роботи.

Проєкт поєднує в собі найкращі практики планування (інтерактивний календар та Kanban-дошка) з потужними інструментами продуктивності (таймер Pomodoro, розумне запобігання конфліктам у розкладі та детальна аналітика).

---

## ✨ Основні можливості

### 🔐 Безпека та профілі користувачів
*   **Локальна авторизація**: Повноцінна реєстрація та авторизація користувачів з ізоляцією даних.
*   **Хешування паролів**: Безпечне збереження паролів у SQLite за допомогою криптографічного хешування.
*   **Менеджер сесій**: Збереження стану авторизації під час роботи програми.

### 📅 Інтерактивний календар & Планувальник
*   **Зручна навігація**: Перегляд завдань на будь-який обраний день за допомогою вбудованого календаря.
*   **Швидке створення завдань**: Створення завдань безпосередньо для вибраної дати.
*   **Візуальна диференціація**: Завдання відображаються з урахуванням пріоритетів та кастомних кольорів.

### 📋 Kanban-дошка
*   **Контроль статусів**: Візуалізація життєвого циклу завдань у розрізі колонок: `To-Do` (До виконання), `В процесі`, `Заблоковано` та `Виконано`.
*   **Гнучке управління**: Зручний інтерфейс для швидкої зміни статусів та аналізу поточного завантаження.

### ⏱️ Розумний трекер часу & Помодоро (Pomodoro)
*   **Інтерактивний таймер**: Відстеження часу в реальному часі зі збереженням робочих сесій у базі даних.
*   **Техніка Pomodoro**: Вбудований режим Pomodoro з автоматичним чергуванням циклів роботи та відпочинку:
    *   *Робота*: 25 хвилин 🍅
    *   *Коротка перерва*: 5 хвилин ☕
    *   *Довга перерва*: 15 хвилин (після кожних 4 робочих циклів)
*   **⚠️ Розумний автозсув & Запобігання конфліктам**: 
    *   При подовженні робочої сесії над завданням система перевіряє, чи не перетинається новий час із пріоритетними справами, які мають фіксований час (`IsTimeLocked`).
    *   Якщо конфліктів немає, застосунок автоматично зсуває час початку для всіх наступних нефіксованих завдань на цей день!
    *   Якщо є перетин із критично важливим завданням, система попередить користувача: *"Краще доробити пізніше, бо у вас заплановані важливіші справи!"*.

### 📊 Детальна аналітика та звіти
*   **Статистика продуктивності**: Розрахунок відсотка виконання та загального часу роботи.
*   **Аналіз часу по днях**: Динамічні стовпчикові діаграми за тиждень або місяць, що показують фактично витрачений час.
*   **Аналітика категорій**:
    *   Рівень виконання завдань у розрізі категорій (тегів).
    *   Топ категорій за витраченим часом.

### 🎨 Кастомізація та теми
*   **ThemeManager**: Повна підтримка темного (**Dark Mode**) та світлого (**Light Mode**) оформлення інтерфейсу.
*   **Гнучкі налаштування завдань**: Можливість задавати пріоритети, категорії (теги), оцінку часу та власні кольори для кожного завдання.

## 🛠️ Технологічний стек

*   **Платформа**: .NET / C#
*   **Інтерфейс користувача (UI)**: WPF (Windows Presentation Foundation) та XAML
*   **Архітектурний патерн**: MVVM (Model-View-ViewModel)
*   **База даних**: SQLite (використовується асинхронна бібліотека `sqlite-net-pcl`)
*   **Керування станом & Behaviors**: Microsoft.Xaml.Behaviors.Wpf (для зв'язування паролів та фокусування елементів)

## 📂 Структура проєкту

Код структуровано відповідно до класичного підходу MVVM:

```text
WpfApp1/
│
├── Core/                   # Базові класи та сервіси
│   ├── ITaskRepository.cs  # Інтерфейс для роботи з даними
│   ├── SqliteTaskRepository.cs # Реалізація репозиторію SQLite
│   ├── SessionManager.cs   # Керування сесіями користувачів
│   ├── ThemeManager.cs     # Керування темами інтерфейсу (Light/Dark)
│   ├── ObservableObject.cs # Реалізація INotifyPropertyChanged
│   └── RelayCommand.cs     # Універсальна реалізація ICommand
│
├── Models/                 # Моделі даних (сутністі БД)
│   ├── User.cs             # Профіль користувача
│   ├── TaskItem.cs         # Завдання (з підтримкою повторюваності)
│   └── TimeItem.cs         # Сесії трекінгу часу
│
├── ViewModels/             # Логіка представлення та стани сторінок
│   ├── MainViewModel.cs    # Головний координатор та навігація
│   ├── LoginViewModel.cs   # Авторизація та реєстрація
│   ├── CalendarViewModel.cs# Планування по днях
│   ├── KanbanViewModel.cs  # Управління статусами завдань
│   ├── TimeTrackerViewModel.cs # Логіка таймера та Pomodoro
│   └── AnalyticsViewModel.cs# Обчислення показників та графіків
│
└── Views/                  # XAML розмітка інтерфейсу
    ├── MainWindow.xaml     # Головне вікно та меню
    ├── LoginView.xaml      # Екран входу
    ├── CalendarView.xaml   # Календарний планувальник
    ├── KanbanView.xaml     # Дошка завдань
    ├── TimeTrackerView.xaml# Трекер часу
    └── AnalyticsView.xaml  # Візуалізація звітів
```

## 🚀 Встановлення та запуск

Для запуску проєкту на локальному комп'ютері виконайте такі кроки:

### 1. Системні вимоги
*   Встановлений **.NET SDK 8.0** або вище.
*   Середовище розробки: **Visual Studio 2022** або **JetBrains Rider**.

### 2. Клонування репозиторію
```bash
git clone https://github.com/grosterr/TimeTrackerApp.git
cd TimeTrackerApp
```

### 3. Відкриття та збірка проєкту
1. Відкрийте файл рішення `Kursova.slnx` або файл проєкту `WpfApp1/WpfApp1.csproj` у вашій IDE.
2. IDE автоматично відновить усі необхідні NuGet-пакети (включаючи `sqlite-net-pcl` та `Microsoft.Xaml.Behaviors.Wpf`).
3. Запустіть проєкт у режимі відладки (**F5** або кнопка Run).

### 4. Робота з базою даних
*   Файл бази даних SQLite створюється автоматично при першому запуску програми.
*   Усі таблиці для користувачів, завдань та сесій часу ініціалізуються автоматично.
